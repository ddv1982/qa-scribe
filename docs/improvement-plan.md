# Improvement Plan

Date: 2026-07-02. Source: full six-track codebase review (Rust core, Tauri shell, React frontend, build/release pipeline, cross-cutting architecture, external best-practice research). Findings reference the code as of v0.4.24.

Guiding goals: correct logic, no DRY violations, simplicity over complexity.

## Phase 1 — Correctness: stop data loss, hangs, and duplicate work

Small, targeted fixes with regression tests. Highest user impact.

1. **Flush autosave before note switch/create/quit** (`frontend/src/app/useAppController.ts:247-256`, `sessionActions.ts:28-58`). Switching or creating a note clears the pending 850ms debounce and silently discards unsaved edits; continuous typing postpones the save indefinitely. `openNote`/`handleNewNote` must await `saveNoteNow()` first, plus a flush on app close. Add a fake-timer regression test (type → switch → assert `updateEntry` called with typed body).
2. **Make `busyAction` reentrancy-safe** (`sessionActions.ts:150-177` vs `generationActions.ts`, `recordActions.ts`). Nested `saveTitle`/`saveBody` clear the outer action's busy state in their `finally`, re-enabling buttons mid-workflow → duplicate AI jobs / duplicate records. Own busy state at the outermost action (counter/set, or pass-through flag). The too-early release of `deletingSessionIdRef` in `handleDeleteNote` (`sessionActions.ts:113-135`) falls out of the same refactor.
3. **Fix process plumbing in provider execution** (`src-tauri/src/commands/ai/provider_execution.rs:157-233`):
   - Register the child in `JobControl` immediately after `spawn()` (closes the un-cancellable window during the prompt write).
   - Start the stderr drain *before* writing stdin; write stdin from its own thread (prevents the classic 64KB pipe-buffer deadlock on large prompts).
   - `wait()` the child on every error path (currently zombies on BrokenPipe / take() failures).
   - Add an overall watchdog timeout on generation.
   - Kill the process *group*, not just the child (AI CLIs spawn node/MCP children; use `process_group(0)` + `killpg` on Unix, Job Objects on Windows), and kill live children on app exit (`RunEvent::ExitRequested`).
4. **Reorder session deletion** (`crates/qa-scribe-core/src/attachments/mod.rs:176-186`): delete the DB row first, files second. Current order permanently deletes evidence files if the DB delete then fails; the reverse only leaves stray files, which `reconcile_attachment_files` already detects.
5. **Make the AI-run failure path infallible and guarded** (`crates/qa-scribe-core/src/services/session_service/generation.rs:118-141`): truncate/normalize the failure message in `fail_ai_run` (a >2000-char provider error currently makes *recording the failure* fail, stranding runs in `running`); add `WHERE status='running'` transition guards; sweep orphaned `running` runs at startup.
6. **Stop passing the Copilot prompt via argv** (`crates/qa-scribe-core/src/ai/mod.rs:159-173`): full session content is visible in `ps` to every local process, and large sessions can exceed ARG_MAX. Use stdin if the CLI supports it; otherwise cap and produce an actionable error. Also whitelist `reasoning_effort` values before interpolating into the Codex `--config` string (`ai/mod.rs:137-141`).
7. **Guard the stream parser's final-text overwrite** (`src-tauri/src/commands/ai/stream_parser.rs:94-112`): any JSON line with a string `result`/`final`/`output` key unconditionally clobbers the accumulated answer. Minimal fix: guard the final path like the snapshot path. Real fix lands in Phase 4 (per-format parsers).
8. **Add one fake-CLI integration test** (small script emitting JSONL) covering: normal completion, mid-stream cancel with kill verification, early-exit EPIPE. The process layer is the most defect-dense and only untested part of the shell.

## Phase 2 — Deletion pass: remove dead weight before refactoring

~30% of the command surface is dead. Deleting first makes every later phase cheaper.

1. **Delete the synchronous generation path**: `generate_ai_action` (`src-tauri/src/commands/ai/action.rs:26-52`), `generate_session_report` (`report.rs`, 133 lines), `run_generation_command` + `From<Output>` (`provider_execution.rs:148-166, 337-347`), and the core report prompt/response code they alone use. This path is unreachable from the UI (which uses `start_ai_action_job`), has no timeout/cancel, and is the worst hang risk in the app.
2. **Prune unused commands**: `get_session`, `get_app_status`, `get_command_shell_status` (+ `implemented_features`/`implemented_commands` lists in `lib.rs`/`status.rs`), `create_generation_context`, `create_ai_run`, `create_evidence_link`, `import_attachment`, `list_attachments`, `get_ai_action_job_status` (or wire it in — see Phase 5), and `export_session` (or wire export into the UI if intended).
3. **Frontend dead code**: the unreachable HTML clipboard renderer (`clipboardExport.ts` `renderHtmlNode`/`renderHtmlImage` ~70 lines + `inlineDataImages`), `serializeEditorHtml`, `persistSettings` return value, the `useSettingsController` mirror-state + hidden write-back effect (`useSettingsController.ts:25-26, 59-77`), unused CSS classes (`record-card*`, `record-grid`, `model-selector`, `provider-hint`).
4. **Electron-era leftovers**: delete tracked `build/entitlements.mac.inherit.plist` (referenced nowhere; also review `allow-jit` entitlement need for WKWebView), prune `out/`/`release/`/`dist-electron/` lines and the no-op `build/` + `!build/` pair from `.gitignore`, delete local clutter (`out/`, `release/`, root `tsconfig.*.tsbuildinfo`, root `node_modules/`, `build/dev-macos-arm64`).
5. **Fix the inert schema versioning** (`crates/qa-scribe-core/src/storage/mod.rs:418` vs `:45`): `user_version` is written 4-then-5 and never read; migrations re-run feature detection (incl. a full `foreign_key_check`) on every open. Either gate on the version properly (consider `rusqlite_migration`, which manages `user_version`) or delete the pragma. Also remove dead code: `render_action_prompt`'s unused `_entries`/`_findings` params, dead `ids.dedup()`, unusable `Deserialize` on `ProviderCapability`, `CopilotRuntime` single-variant enum.

## Phase 3 — DRY consolidation within layers

1. **One Rust `html` utility module** shared by `generation/{prompt,response,html_projection}.rs`: the verbatim-duplicated attribute parser + `skip_whitespace` (~90 lines ×2, plus a third weaker scanner), the two overlapping entity decoders, `escape_html_attribute` (byte-identical ×2), and `MANAGED_ATTACHMENT_PROTOCOL` (defined ×3 in core, ×4 counting TS). Fix the O(n²) `to_ascii_lowercase` scans while there.
2. **Storage/service helpers**: `with_immediate_tx` on `Database` via `unchecked_transaction()` (collapses 3 hand-rolled BEGIN/COMMIT/ROLLBACK blocks, RAII rollback); wrap `update_entry`/`update_finding`/`update_draft` read-then-write in transactions; one `require_owned_by_session(table, id, session_id)` helper with `.optional()` → `NotFound` (replaces 4 ad-hoc shapes; add the missing session check in `create_entry`); named constants for the repeated limits (160 title, 40 body_format, 2000/240/120/500 session fields); dedupe the double ownership checks in `import_managed_attachment` → `_bytes`.
3. **Table-driven provider detection** (`src-tauri/src/commands/providers/detection.rs:67-193`): `detect_claude`/`detect_codex` are line-for-line clones modulo five strings; move `auth_args`/install hint/ready command onto `ProviderCapability`. Make "Refresh" rebuild the PATH snapshot (`provider_command.rs:13-14` OnceLock is never invalidated) and stop letting a cached Deep failure outrank a working Fast result.
4. **Frontend consolidation**: single `editor/htmlUtils.ts` for `escapeHtml` (×4), `escapeAttribute` (×2), `isSafeUrlWithProtocols` (×2), `managedAttachmentIdFrom*` (×3) — these are security-relevant and must not fork; generic `RecordCollectionView` replacing the ~95% clone pair TestwareView/FindingsView (~150 lines + the 20-prop lists in AppShell); parameterize the triplicated Jira-copy handlers and quadruplicated create handlers; import `StoredRichBody` instead of re-declaring.
5. **Scripts/release constants**: one constants module for bundle ID, Pages URL, setup-package filename, DMG basename (currently spelled in 4 languages); reuse `command-utils.mjs` helpers instead of per-script reimplementations; **add a `bump` script** as the single write-path for the 7 version locations, keeping `check-release-metadata.mjs` as the guard.

## Phase 4 — Cross-boundary structure

1. **Adopt tauri-specta codegen** (rc-versioned but the de facto standard for Tauri 2; specta's TS exporter is marked stable). `#[specta::specta]` per command, `derive(specta::Type)` on the ~40 boundary types, export bindings on debug startup; diff against the hand-written bridge, then delete `tauriTypes.ts` and most of `tauriCommands.ts`. While migrating, **decide the Patch-null contract explicitly**: TS currently advertises `title?: string | null` where `null` is a silent no-op in Rust's double-`Option` (`SessionPatch`, `EntryPatch.body`, `FindingPatch`, `DraftPatch`). Also single-source the provider model/reasoning defaults duplicated between `domain/settings.rs:68-82` and `frontend/src/settings/defaults.ts`.
2. **Structured boundary errors**: replace `Result<T, String>` with serialized `CommandError { kind, message }` mapped from `QaScribeError`; the frontend's `formatError` prose-matching and the backend's stderr keyword matching stop being the only classification mechanism. ~30 mechanical call sites.
3. **Move the AI generation workflow into core** behind a `ProviderExecutor` trait: `prepare/finish_ai_action_generation`, prompt versions (`types.rs:43-49`), testware preferences (`preferences.rs`), evidence-link creation, title derivation, and the stream parsers (reuniting `GenerationOutputFormat` with its consumer). src-tauri keeps process spawning, PATH, jobs, Channel plumbing. Then extend the `qa-scribe-app` smoke harness with a fake executor so the flagship feature is exercised headlessly.
4. **Per-format stream parsers** (Claude stream-json, Codex JSONL) keyed on explicit event types, replacing the universal heuristic sniffing (root cause of the final-text clobber).
5. **Domain language alignment with CONTEXT.md**: frontend "record" (unsanctioned umbrella for Draft|Finding — sanction it or rename), "note" used for Session (`NotesView`, `handleDeleteNote`, "Untitled note" — Note is defined as an Entry), `WorkspaceView` ("Workspace" is on the Avoid list). Single-source the editor-HTML tag allowlist + attachment protocol from Rust via codegen so `editorHtml.ts` derives instead of restating.

## Phase 5 — Hardening, performance, release robustness

1. **Sanitization defense-in-depth** (per current Tauri guidance: XSS in a webview = IPC escalation): close the `bodyJson` path that bypasses the HTML sanitizer (`editorDocument.ts:26-30`); add DOMPurify at render for any non-editor HTML injection; consider ammonia Rust-side on ingest of AI output; evaluate the Isolation pattern (now officially "highly recommended") and `tauri_build::AppManifest::commands` allowlisting.
2. **Attachment integrity**: validate sha256 as exactly-64 lowercase hex; verify hashes on read (or drop the column — currently pure overhead); fix filename-length validation vs on-disk uuid prefix; fix duplicate-filename ambiguity in `restore_known_attachment_sources`; replace free-form `import_attachment` source paths with the dialog-plugin picker pattern (arbitrary-file-read primitive if the frontend is ever compromised).
3. **Schema**: indices on cascade FK columns (`evidence_links.*`, `generation_context_*`, `drafts.ai_run_id`, `attachments.entry_id`); UTF-8 char counts instead of byte counts in validation messages.
4. **Frontend performance + resilience**: `useMemo` for `testwareDrafts` and `noteWordCount` (currently DOMParser/generateHTML passes per keystroke); split `useAppController` (god-hook, ~50 members, full-app re-render per keystroke); wire `get_ai_action_job_status` into boot to reconcile orphaned generation jobs after a webview reload; verify `window.prompt` works in packaged wry for the Link button (documented no-op on some platforms — jsdom test passes regardless); switch eslint to `recommendedTypeChecked` (`no-floating-promises` would catch the Phase 1 bug class); Escape/focus-trap for modals.
5. **CI/release**: add Intel macOS coverage to CI (or at least `cargo check --target x86_64-apple-darwin`) — currently first compiled at release time; run `release:check` in CI (the Python test suite is currently never executed by any workflow); guard the APT Pages deploy against publishing an older version than live (tag re-run currently clobbers the repo last-writer-wins); publish signed SHA256SUMS for all release assets (mechanism 90% exists for the setup .deb); install tauri-cli prebuilt instead of compiling ~10min ×3 jobs; align `bun run verify` with the CI gate (audit + metadata checks); decouple Linux packaging from the full matrix result.

## Suggested sequencing

Phases 1-2 are independent of each other and can land as a series of small PRs. Phase 3 before Phase 4 (consolidation shrinks the codegen/move surface). Phase 5 items are largely independent and can be interleaved. Every phase should keep `bun run verify` green and preserve Tauri command names, storage schema, and product language per docs/code-size-guidelines.md.
